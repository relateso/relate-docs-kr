## Overview
Relate Calendar(Beta)는 캘린들리(Calendly), 되는시간과 같은 미팅 스케줄링용 캘린더 제품으로, 이를 활용해 잠재고객과 고객이 쉽고 빠르게 미팅 일정을 잡을 수 있습니다. 

## Relate Cal 시작하기

### 1/ 구글/아웃룩 연동 체크
Relate Cal을 사용하기에 앞서, 구글/아웃룩 연동 여부를 꼭 체크해야 합니다. Relate에 고객과 구글/아웃룩에서 주고받은 이메일이 캘린더에 동기화되어 있지 않다면, Meeting Scheduler를 생성할 수 없기 때문입니다. 

구체적인 이메일 연동 과정은 [이 링크](https://www.relate.kr/docs/setup/email-sync#3)에서 확인할 수 있습니다. 

### 2/ Meeting Scheduler 생성 
이메일 연동 후, Relate 메뉴 왼쪽에 있는 Meetings 탭을 선택합니다. 

다음, 상단의 ‘+ Meeting Scheduler’ 버튼을 클릭하면 아래와 같은 팝업창이 뜨게 됩니다.

![cal2](/images/products/cal/c2.png)

### 2-1/ Hosts 
Host 섹션에서 1개의 미팅 캘린더에 대해 1명 이상의 호스트를 추가할 수 있습니다. 
다음은 Relate Calendar에서 호스트 명수와 순환 여부에 따른 다양한 use-case에 대한 설명입니다: 

1. 싱글 호스트(Single Host): 잠재고객이나 고객과의 1:1 미팅에 가장 적합한 방식  
2. 라운드 로빈(Round Robin) - 순환 배정: 여러 명의 호스트들을 순환하며 미팅을 배정하는 방식으로, 각 호스트는 일정한 비율로 미팅 스케줄을 나눠받음  
3. 라운드 로빈(Round Robin) - Max Availability: 위 라운드 로빈과 동일하나, 일정한 비율로 배정 받는 것이 아닌, 고객에게 가능한 시간대를 모두 보여줌으로서 최대한 많은 슬롯을 스케줄링하는 시나리오 

### 2-2/ Location 
호스트 지정이 끝나면, Google Meet 이나 Zoom에서 온라인 미팅룸 URL을 생성하여 Scheduler의 Location 항목에 회의 링크를 추가해줍니다. 

이를 통해 캘린더에도 해당 링크가 자동적으로 삽입되어, 손쉽게 사용자별로 개별적인 고유 미팅룸을 스케줄링할 수 있습니다. 

뿐만 아니라 방문 미팅용 링크를 따로 만들고 ‘방문 미팅'이라고 입력한 후, 추후 캘린더에서 수정하는 방식으로 활용할 수도 있습니다. 


### 3/ 다양한 기능 살펴보기 
Relate Calendar는 현재 아래와 같이 다양한 기능들을 지원합니다.

### 3-1/ Event Info

![cal3](/images/products/cal/c3.png)

Event Info 페이지에서는 미팅의 길이나 간격 등 세부 사항들을 커스터마이징 할 수 있습니다: 

- Duration: 미팅 길이
- Min. Booking Notice: 미팅을 잡기 위해 최소 몇 시간/일 전에 잡아야 하는지
- Min. Cancel Notice: 미팅 취소 전 최소 몇 분/시간/일 전에 알려줘야 하는지
- Future Limit: 앞으로의 몇 달/주/일 기간만 보여줄 것인지
- Buffer Time: 캘린더에 다른 일정이 있을 경우, 앞 뒤 일정과 얼마나 간격을 둘 것인지
- Cancellation Policy: 미팅을 취소하는 사용자에게 어떤 메시지를 보여주고 이유를 물어볼 것인지

### 3-2/ Calendars 

![cal4](/images/products/cal/c4.png)

Calendars에서 미팅 가능 시간대와 일정의 기준이 될 캘린더 계정 이메일을 지정해줍니다. 

### 3-3/ Opening Hours

![cal5](/images/products/cal/c5.png)

Opening Hours에서 고객에게 보여질 가능 시간대를 설정할 수 있습니다. 기본적으로 유저의 캘린더 중 빈 시간만 보여지지만, 특정 일자나 시간대를 옵션에서 제외하고 싶을 경우 수정할 수 있습니다. 

### 3-4/ Booking Flow

![cal6](/images/products/cal/c6.png)

Booking Flow에서 새로운 미팅 요청이 들어왔을 때의 수락 방식과 이동 페이지를 설정할 수 있습니다: 

- Automatic: 일정이 생기면 자동으로 수락
- Manual: 일정이 생겼을 때 사용자가 수락 여부를 추후 표시하도록 함
- Hosted Page/Custom Page: 미팅 예약 완료 후 특정 페이지로 이동하도록 설정 가능

### 3-5/ Reminders

![cal7](/images/products/cal/c7.png)

Reminders에서 미팅 대상자에게 미팅 컨펌/미팅 시작 전 리마인드 이메일을 전송할지, 전송한다면 어떠한 문구로 리마인더를 보낼지 설정할 수 있습니다. 

### 3-6/ Custom Fields

![cal8](/images/products/cal/c8.png)

Custom Fields 페이지에서 미팅을 신청하고자 하는 게스트가 제공해야 하는 정보들을 지정합니다. 
필수로 제공해야 하는 이름과 이메일 이외에도, Label과 Type 등의 정보들을 추가 요구할 수 있습니다. 

### 3-7/ Page Styles

![cal9](/images/products/cal/c9.png)

Page Styles에서 회사명, 로고, BI 컬러 등 디테일적 요소들을 설정할 수 있습니다. 
설정이 끝난 후, 하단의 ‘View Page’ 버튼을 클릭하여 미팅 페이지 UI를 체크합니다.
